# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in OpenSDD, please report it privately.

**Preferred**: Use [GitHub's private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) for this repository.

**Alternative**: Email the maintainer directly (see commit history for contact) or open a public issue for non-sensitive concerns.

We'll respond within 48 hours and work to address the issue promptly.
